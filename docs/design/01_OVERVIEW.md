# 01. 概要

## 目的
Laravel 12（PHP 8.5）でブログサービスを構築する。
- 非登録ユーザ：記事の閲覧・検索
- 登録ユーザ：記事投稿・編集・削除、コメント、タグ付け、お気に入り

初期リリースは最小構成（MVP）で実装し、将来のAPI化・SPA化を阻害しないシンプルな設計を採用する。

## 前提
- フレームワーク：Laravel 12
- テンプレート：Blade
- DB：MySQL想定
- 開発環境：Laravel Sail + Docker
- コーディング規約：PSR-12
- UseCaseパターン採用（Controllerにビジネスロジックを置かない）
- DDDは採用しない（過度に複雑化させない）

## MVP範囲（基本機能）
- 認証：登録・ログイン・ログアウト・パスワードリセット（メール認証はオプション）
- 記事：一覧（新着/人気）・詳細・作成・編集・削除（論理削除）・下書き
- 検索：タイトル/本文/タグ、ページネーション
- コメント：投稿・編集（本人）・削除（本人または記事投稿者）
- お気に入り：登録/解除・自分のお気に入り一覧・お気に入り数表示
- タグ：記事に紐づくタグ作成/既存タグ再利用、slugで一意

## 非機能
- 応答：通常アクセス1秒以内を目標、N+1回避
- セキュリティ：CSRF/XSS/SQLi対策（Laravel標準）、Policy徹底、bcrypt
- 拡張性：将来のJSON API提供を見据えた責務分離
