# 06. 画面設計（UI/UX）

方針：シンプル、情報密度は控えめ、操作導線を明確化。
CSSは Tailwind または Bootstrap を採用（どちらでも成立する作りにする）。

## 共通レイアウト
- ヘッダー：ロゴ、検索ボックス、ログイン状態のメニュー
- フッター：コピーライト
- フラッシュメッセージ：成功/失敗

## 画面一覧

### 1) トップ/記事一覧（/、/posts）
表示：
- 記事カード：タイトル、抜粋、投稿者名、投稿日、タグ、favorites_count、comments_count
- 並び替え：新着/人気
- ページネーション
- フィルタ：タグ、検索キーワード

操作：
- 記事詳細へ遷移
- （ログイン時）新規投稿ボタン（/posts/create）

### 2) 記事詳細（/posts/{post}）
表示：
- タイトル、本文、タグ、投稿者、投稿日
- favorites_count、view_count（必要なら表示）
- コメント一覧（投稿者、本文、日時）
- コメント投稿フォーム（ログイン時）

操作：
- （本人）編集/削除
- （ログイン時）お気に入り登録/解除（トグル）
- （コメント）本人は編集、本人または記事投稿者は削除

### 3) 記事投稿/編集（/posts/create, /posts/{post}/edit）
入力：
- title（必須）
- body（必須）
- tags（任意：カンマ区切り等）
- status（draft/published）

操作：
- 下書き保存（status=draft）
- 公開（status=published）

UXメモ：
- タグ入力は最初はテキストで十分（補完は将来）

### 4) ログイン/登録
Laravel標準UIで最小構成。
パスワードリセットも標準を利用。

### 5) マイページ（/mypage）
表示タブ：
- 自分の投稿一覧（draft/publishedの区別）
- お気に入り一覧
